**`handle_llm_tab(request)`** — GET handler for `/settings/llm`:
- Import `load_config` from `hermes_cli.config`, `is_managed` from `hermes_cli.config`, `get_env_value` from `hermes_cli.config`, `redact_secret` from `gateway.dashboard.auth`, `PROVIDER_REGISTRY` from `hermes_cli.auth`
- All imports INSIDE the function body (deferred import pattern — matches auth.py existing pattern, avoids import-time failures when hermes_cli not available)
- `config = load_config()`
- Parse model string: `provider, _, model_name = config.get("model", "").partition("/")`
- Build provider list: iterate `PROVIDER_REGISTRY`, include `name`, `display_name`, `auth_type`, first `api_key_env_vars` entry (or empty string)
- For current provider, get the API key env var from registry, call `get_env_value(env_var)`, redact with `redact_secret()` for display
- Read `agent.temperature` and `agent.max_tokens` from config with defaults (empty string if not present)
- Pass `managed = is_managed()` to template
- Return `aiohttp_jinja2.render_template("settings/_llm.html", request, context)` (synchronous — do NOT await, per Phase 1 lesson)

**`handle_llm_save(request)`** — POST handler for `/settings/llm/save`:
- Check `is_managed()` first — if True, return the LLM template with `flash="error:Configuration is managed by NixOS/Enclava. Changes must be made through your system configuration."`
- `data = await request.post()`
- Inside `async with request.app["config_lock"]:` block:
  - `config = load_config()`
  - Read `provider = data.get("provider", "").strip()` and `model_name = data.get("model_name", "").strip()`
  - If both present: `config["model"] = f"{provider}/{model_name}"`; if only model_name: `config["model"] = model_name`
  - Read `temperature = data.get("temperature", "").strip()` — if non-empty, set `config.setdefault("agent", {})["temperature"] = float(temperature)`
  - Read `max_tokens = data.get("max_tokens", "").strip()` — if non-empty, set `config.setdefault("agent", {})["max_tokens"] = int(max_tokens)`
  - Call `save_config(config)`
- After the lock block, handle API key separately:
  - `api_key = data.get("api_key", "").strip()`
  - `api_key_env_var = data.get("api_key_env_var", "").strip()`
  - If `api_key` and `"***" not in api_key` and `api_key_env_var`: call `save_env_value(api_key_env_var, api_key)`
- Re-render the LLM tab with `flash="success:LLM settings saved."` by calling the same render logic as handle_llm_tab but with flash set

def read_soul_md() -> str:
    from hermes_constants import get_hermes_home
    soul_path = Path(get_hermes_home()) / "SOUL.md"
    if soul_path.exists():
        return soul_path.read_text(encoding="utf-8")
    return ""

def write_soul_md(content: str) -> None:
    from hermes_constants import get_hermes_home
    soul_path = Path(get_hermes_home()) / "SOUL.md"

Create `gateway/dashboard/templates/settings/_llm.html` — this is a partial template (NOT extending base.html) loaded via HTMX into `#tab-content`.

The template receives these context variables from `handle_llm_tab` / `_render_llm`:
- `provider` — current provider string (e.g., "anthropic")
- `model_name` — current model name (e.g., "claude-opus-4.6")
- `api_key_display` — redacted API key string (e.g., "sk-a***yz") or empty
- `api_key_env_var` — env var name for current provider's API key (e.g., "ANTHROPIC_API_KEY")
- `providers` — list of dicts: `[{"name": "anthropic", "display_name": "Anthropic", "auth_type": "api_key", "env_var": "ANTHROPIC_API_KEY"}, ...]`
- `temperature` — current temperature value or empty string
- `max_tokens` — current max_tokens value or empty string
- `managed` — boolean
- `soul_content` — string content of SOUL.md
